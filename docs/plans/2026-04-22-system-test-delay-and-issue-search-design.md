# 系统测试模块补齐设计

## 背景

系统测试模块下仍缺少两个可落到现有抽象链路的页面：

- 申请延期缺陷分析
- 议题查询

本次目标是优先复用现有组件和服务抽象，不新增专用表格框架，不影响已完成的系统测试缺陷汇总、议题阶段统计、缺陷原因分析。

## 方案

### 申请延期缺陷分析

- 继续复用 `StatisticBoardPage -> StatisticBoardView -> BaseStatisticTable`
- 后端复用现有统计板注册机制，新增 `system-test-delay-analysis`
- 行维度为延期原因
- 列为一级缺陷、二级缺陷、三级缺陷、建议类、总计
- 筛选项先保留老平台核心能力：测试阶段
- 单元格下钻复用现有统计板明细抽屉和导出能力

### 议题查询

- 继续复用 `BaseRecordTable`
- 新增一个系统测试议题查询页面，不自建专用筛选布局
- 后端基于 `issue_fact` 现有稳定字段提供查询和筛选项接口
- 第一版只实现老平台核心能力：筛选、分页、排序、展开详情
- 导出暂不实现，避免为第一版引入额外复杂度

## 数据边界

- 仅使用新平台 `issue_fact` 已有字段
- 不补造老平台缺失的细粒度原因/功能字段
- `议题查询` 第一版不包含新平台事实层中不存在的字段，例如功能名称

## 风险控制

- 路由层只替换对应占位页，不改其他已上线页面
- 统计板只新增映射，不修改现有 board key
- 议题查询接口单独命名，避免和其他记录页接口耦合
- 所有改动补充最小测试和构建验证
