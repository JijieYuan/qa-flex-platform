# 前端记录页收口规则

本文档只约束非看板、非外部数据来源的记录页和分析页，避免筛选、分页、导出和明细逻辑继续分散。

## 收口范围

优先复用共享底座：

- 查询参数同步
- 条件筛选
- 表格分页
- CSV 导出
- 明细抽屉
- 规则说明
- 错误提示和加载状态

暂不强行抽象：

- 看板页面
- 外部数据来源页面
- 产品形态仍未稳定的特例分析页
- 需要大量定制布局的专题页面

## 新增记录页规则

新增非看板记录页时，默认按以下顺序实现：

1. 先确认后端请求对象和响应字段。
2. 再接入共享 `api-client`。
3. 优先使用共享页面控制器或共享条件筛选能力。
4. 表格展示只保留页面自己的列定义和字段格式化。
5. 导出、分页、筛选、明细抽屉不重复造一套。
6. 补充挂载测试和关键交互测试。

## 特例页面退出条件

特例页面可以暂时保留，但需要满足以下任一条件后再迁入共享底座：

- 查询、明细、导出三类接口契约已经稳定。
- 页面已经出现第二处相同筛选或分页逻辑。
- 新增需求需要复用已有记录页能力。
- 测试补齐后能稳定保护现有行为。

## 检查清单

修改记录页时逐项确认：

1. 是否新增了重复的 query 解析逻辑。
2. 是否新增了重复的分页状态管理。
3. 是否新增了重复的 CSV 导出逻辑。
4. 是否绕开了 `api-client`。
5. 是否绕开了已有条件筛选字段定义。
6. 是否补充或更新了挂载测试。

## 自动守卫

提交前运行：

```powershell
python scripts/check_frontend_api_boundary.py
```

页面、组件和组合式函数不应直接写 `/api/**` 字符串或调用 `fetch`。真实请求统一放到 `frontend/src/api-client`，测试文件可以保留 mock URL。
