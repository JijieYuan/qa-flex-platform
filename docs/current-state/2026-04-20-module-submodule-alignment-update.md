# 主模块与副模块结构对齐更新
更新时间：2026-04-20
适用项目：`D:\projects\data_collection_platform`

## 1. 背景

当前新平台已经完成了主模块一级结构：

- 统计分析
- 评审数据
- 代码走查
- 集成测试
- 议题统计
- 客户问题
- 系统设置

但和老平台相比，副模块入口还不完整。老平台在这些主模块下面已经形成了比较稳定的二级入口结构，例如：

- 代码走查下有“非法数据”和“多元看板”
- 议题统计下有“系统测试缺陷汇总”“议题多元看板”“申请延期缺陷分析”“议题查询”等
- 客户问题下也有一组更完整的统计与查询入口

本轮的目标不是一次性把所有副模块都实现成正式业务页面，而是先把**平台结构骨架补齐**。

## 2. 本次调整

本次主要调整：

- `frontend/src/navigation.ts`
- `frontend/src/router.ts`

处理原则：

1. 老平台已经存在、且新平台已经有真实能力支撑的页面，继续指向真实页面
2. 老平台已存在、但新平台当前还没有真实页面和后端支撑的副模块，统一先接入 `ModulePlaceholderView`
3. 不新增重型页面壳，不破坏现有主模块结构
4. 先把“导航和路由的结构完整性”做对，再分批填充真实业务内容

## 4. 当前边界

这次完成的是：

- 主模块与副模块的导航结构补齐
- 路由入口补齐
- 当前真实能力与老平台菜单结构的第一轮映射

这次没有完成的是：

- 老平台所有副模块的真实业务复刻
- 新增统计板服务
- 新增记录页/查询页后端接口
- 新增导出、规则说明、详情下钻等页面能力

也就是说，本轮完成的是**平台模块骨架对齐**，不是**业务能力全部补齐**。

## 5. 结论

现在新平台已经具备一套更完整的主模块/副模块层级结构：

- 顶层主模块对齐当前产品架构
- 二级副模块尽量对齐老平台习惯
- 已有真实页继续保留
- 缺失页面先统一走轻量占位

后续如果继续推进，可以按模块逐步把这些占位页替换成真实记录页、统计页或查询页，而不需要再反复调整平台导航结构。
