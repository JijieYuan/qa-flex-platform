# 2026-04-10 规则编辑模块重构更新

## 背景

代码走查非法记录页的规则编辑区，之前经历了多轮 Demo 演进，但页面结构逐渐偏向“说明式配置器”与“重卡片交互”，和当前项目想要的轻量后台体验不一致。

这轮调整的目标很明确：

- 彻底重构规则编辑模块的表现层
- 保留现有功能边界不变
- 不改规则说明模块的展示与口径
- 视觉和交互回到更接近 Element Plus 原生与轻阿里系后台的方向

## 本轮改动

### 1. 上半区规则编辑模块整体重做

位置：

- `frontend/src/views/CodeReviewIllegalRecordsView.vue`
- `frontend/src/components/rule-config/RuleExpressionEditor.vue`

新的交互结构改为：

- 顶部仅保留标题、预览开关、新增条件组、恢复默认
- 中部按“条件组”组织筛选结构
- 每个条件组单独展示命中条数与删除动作
- 条件组之间用轻量的 `或` 分隔
- 组内关系通过小型 `且 / 或` 切换器表达
- 具体条件使用单行 Element Plus 风格输入控件表达

### 2. 去掉旧版重卡片与说明式冗余

这一轮明确摒弃了上一版的这些表现：

- 过重的说明块与大面积背景块
- 旧的 `rule-demo-*` 与 `rule-refactor-*` 交互样式
- 额外的句式说明和重复解释文字
- 规则编辑区内部重复的预览说明结构

### 3. 保留功能不变

本轮没有改动规则编辑器的功能边界，仍然保留：

- 条件组增删
- 组内条件增删
- 嵌套分组
- `AND / OR` 逻辑组合
- 预览开关
- 命中数预览
- 命中样本查看
- 本地状态持久化
- 恢复默认

## 没有改动的部分

本轮明确没有动下面这些内容：

- 后端正式规则说明接口
- 规则说明与统计过程展示区
- 规则说明的 Flow、指标定义、正式口径说明
- 后端真实非法判定逻辑

也就是说，这次只重构了“规则编辑模块”，没有改变“规则说明模块”。

## 视觉方向

本轮 UI 风格收口原则：

- 尽量贴近 Element Plus 原生后台风格
- 用细边框、浅背景、轻分隔替代厚卡片
- 控件行内化，提升一屏信息密度
- 仅在需要时用少量平台蓝作为状态强调
- 整体配色继续沿用当前网页壳的灰白 + 深色文字体系
