# 评审数据管理种子数据与交互补充

更新时间：2026-04-17  
适用项目：`D:\projects\data_collection_platform`

## 本次补充目标

本轮不是重做评审数据管理页，而是在现有独立评审域方案上补足三件影响体验的事情：

1. 本地无数据时，页面看起来像“空页”
2. 页面缺少手动刷新入口
3. QA 首次打开弹窗时，不容易理解“先做什么、后做什么”

## 变更内容

### 1. 新增本地种子脚本

新增脚本：

- `scripts/seed-local-review-data-management-demo.sql`

脚本会补充三类演示数据：

- `review_records`
- `review_record_experts`
- `review_problem_items`

数据内容按真实用户视角组织，覆盖：

- 多条评审主记录
- 每条记录下的多个评审专家
- 已修复 / 新提出 / 已拒绝 等不同问题状态
- 可直接验证主表、展开子表、编辑和删除流程

### 2. 恢复刷新按钮

评审数据管理页恢复页面级刷新入口，便于：

- 刚执行完 seed 后立刻重新拉取数据
- 手动新增、编辑、删除后快速刷新列表与筛选项
- 降低 QA 对“页面为什么没变化”的困惑

### 3. 在对话框内补使用说明

为了降低首次使用门槛，本轮在两个弹窗顶部加入简洁提示：

- 新建/编辑评审弹窗  
  提示“先建主记录，再在展开行里维护问题清单”

- 新建/编辑问题弹窗  
  提示“先选评审专家和状态，再补位置、描述和建议方案”

说明控制在一行，不额外增加培训成本，也不引入重说明式 UI。

## 当前推荐使用方式

1. 执行 `scripts/seed-local-review-data-management-demo.sql`
2. 打开 `/#/review-data/home`
3. 点击页面上的“刷新”
4. 先通过“新增评审”录入主记录
5. 再展开某条记录，在“评审问题清单”里继续新增、编辑、删除

## 本轮结论不变

- 评审数据管理继续使用独立评审域表
- 当前不切回 `collect_form_records`
- 镜像库仍然只适合作为辅助候选来源，不作为评审数据管理主来源
- 后续若继续贴近老平台真实使用方式，更合适补的是“导入入口”，而不是改成镜像驱动
