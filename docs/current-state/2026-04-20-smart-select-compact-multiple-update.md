# SmartSelect 紧凑多选优化
更新时间：2026-04-20
适用项目：`D:\projects\data_collection_platform`

## 1. 调整背景

评审数据管理页的“新增评审”弹窗中，`评审专家` 使用的是 `SmartSelect` 的紧凑多选模式。
原先虽然已经支持首字母搜索和多选，但下拉项仍然接近“一行一个”的单选式排布，短姓名没有充分利用横向空间。

## 2. 本次调整

文件：
- `frontend/src/components/base/SmartSelect.vue`
- `frontend/src/views/review-data/ReviewRecordFormDialog.vue`

本次保留 `Element Plus el-select` 作为基础能力，不新增重型自定义选择器，只对 `compact + multiple` 场景做样式分流：

- 紧凑单选继续沿用原有轻量胶囊列表样式
- 紧凑多选切换为固定面板宽度下的自适应流式布局
- 下拉面板宽度跟随输入框，不允许被选项内容反向撑宽
- 短选项尽量按内容宽度收紧，能放三项就放三项，长选项再自然换行
- 选项块内部统一做居中对齐，避免中文短姓名视觉上偏左、留白失衡
- 因此中文短姓名可以一行容纳三个甚至更多，英文较长姓名通常保持两项一行
- 不影响当前其他单选筛选器和普通下拉

## 3. 调整结果

现在像“评审专家”这样的多选下拉：

- 会比原先更紧凑
- 横向利用率更高
- 短人名不会再默认一行只占一个选项位
- 长一点的选项不会把整行撑满或把面板撑宽
- 仍然保留首字母 / 拼音首字母搜索能力

## 4. 设计取舍

这次没有改成完全自定义的 `Popover + 面板多选器`，原因是：

- 当前需求主要是优化布局密度，不是重做交互模型
- 继续基于 `SmartSelect + Element Plus` 更稳
- 复用性更好，后续其他页面的紧凑多选也能直接受益
- 维护成本明显低于新做一套多选组件

## 5. 当前结论

`SmartSelect` 现在已经区分了：

- 紧凑单选
- 紧凑多选

后续如果平台里再出现“人员选择、多标签选择、短枚举多选”这类场景，可以优先复用这套紧凑多选样式，而不必重复开发新的下拉组件。
